## 快速开始 🚀

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置API密钥

复制并编辑环境变量文件：

```bash
cp .env.example .env
```

在 `.env` 文件中填入您的 Lighter API 密钥：

```
LIGHTER_API_KEY=your_api_key_here
LIGHTER_API_SECRET=your_api_secret_here
```

### 3. 运行机器人

```bash
# 使用默认配置运行
python main.py

# 指定策略运行
python main.py --strategy grid

# 模拟模式（不实际下单）
python main.py --dry-run

# 设置交易量目标
python main.py --volume-target 200000

# 查看所有选项
python main.py --help
```

## 策略说明 📖

### 自成交策略 (Self Trade)
- **原理**：同时下买卖单，通过价格交叉实现快速成交
- **优点**：交易量产生速度快
- **缺点**：成本较高，需要注意风险控制
- **适用场景**：需要快速完成交易量目标

### 网格策略 (Grid)
- **原理**：在价格上下布置多层订单，等待市场波动成交
- **优点**：成本低，可以赚取价差
- **缺点**：依赖市场波动，完成速度不确定
- **适用场景**：市场波动较大时效果最佳

### 做市策略 (Market Make)
- **原理**：同时提供买卖流动性，赚取买卖价差
- **优点**：可获得手续费返佣，长期盈利潜力
- **缺点**：需要较大资金，有库存风险
- **适用场景**：适合长期运行，追求稳定收益

### 混合策略 (Hybrid)
- **原理**：根据市场情况自动切换不同策略
- **优点**：适应性强，综合多种策略优势
- **缺点**：逻辑复杂，需要更多调试
- **适用场景**：不确定市场状况时的最佳选择

## 配置说明 ⚙️

主要配置项在 `config.py` 中：

```python
# 交易参数
MIN_ORDER_SIZE = 10        # 最小订单量（USDT）
MAX_ORDER_SIZE = 100       # 最大订单量
DAILY_VOLUME_TARGET = 100000  # 日目标交易量

# 风险控制
MAX_DAILY_LOSS = 50        # 最大日亏损（USDT）
MAX_POSITION_SIZE = 1000   # 最大持仓
STOP_LOSS_PERCENT = 2      # 止损百分比

# 策略参数
GRID_LEVELS = 10           # 网格层数
GRID_SPACING_BPS = 20      # 网格间距（基点）
```

## 安全建议 🔒

1. **API密钥安全**
   - 永远不要分享您的API密钥
   - 使用只有交易权限的API密钥
   - 定期更换密钥

2. **风险控制**
   - 设置合理的最大亏损限制
   - 不要使用全部资金
   - 定期检查账户状态

3. **运行环境**
   - 使用稳定的网络连接
   - 建议在VPS上运行
   - 设置进程守护

## 常见问题 ❓

**Q: 如何获取Lighter API密钥？**
A: 访问 [Lighter官网](https://lighter.xyz) 注册账户并在API管理页面创建密钥。

**Q: 运行需要多少资金？**
A: 建议准备至少 1000 USDT，更多资金可以提高效率。

**Q: 哪种策略最好？**
A: 取决于市场情况。高波动时用网格，低波动时用做市，追求速度用自成交。

**Q: 如何降低成本？**
A: 使用Maker订单类型，选择网格或做市策略，避免频繁市价单。
